1. What is a Graph?

A graph is a non-linear data structure consisting of a set of vertices (nodes) connected by edges (links). Formally, a graph G = (V, E) where V is the vertex set and E ⊆ V × V is the edge set.

Unlike trees, graphs can have cycles, multiple edges between nodes, and disconnected components. They model practically anything where things relate to other things — friendships, road networks, dependencies, web pages, molecules, neural networks.

Why graphs matter: any problem that can be reframed as "nodes and connections" instantly inherits decades of optimal algorithms. Recognizing the graph structure is often half the solution.

3. Essential Terminology

  • Degree: number of edges incident to a vertex. In digraphs: in-degree and out-degree.
  • Path: sequence of vertices connected by edges with no repeats.
  • Walk: sequence allowing repeats. Trail: no repeated edge.
  • Cycle: path that starts and ends at the same vertex.
  • Subgraph: graph formed from a subset of V and E.
  • Component: maximal connected subgraph.
  • Spanning Tree: tree subgraph touching every vertex.
  • Diameter: longest shortest-path between any two vertices.
  • Girth: length of shortest cycle.
  • Adjacent: two vertices sharing an edge.
  • Self-loop: edge from a vertex to itself.

4. Graph Representations

Adjacency Matrix

2D array where M[i][j] = 1 (or weight) if edge exists. Space: O(V²). Edge lookup: O(1). Best for dense graphs.

int n = 5;
int[][] adj = new int[n][n];
adj[0][1] = adj[1][0] = 1;   // undirected edge 0-1
adj[1][2] = adj[2][1] = 1;

Adjacency List

Array of lists; adj[u] stores neighbors of u. Space: O(V + E). Best for sparse graphs (most real-world graphs).

// int[] {neighbor, weight}
List<List<int[]>> adj = new ArrayList<>();
for (int i = 0; i < n; i++) adj.add(new ArrayList<>());
adj.get(0).add(new int[]{1, 4});   // edge to 1 with weight 4
adj.get(1).add(new int[]{0, 4});

Edge List

Just a list of (u, v, w) tuples. Compact; used by Kruskal & Bellman-Ford.

// int[] {u, v, weight}
int[][] edges = { {0, 1, 4}, {1, 2, 3}, {0, 2, 5} };

5. Breadth-First Search (BFS)

Explores level by level using a queue. Finds the shortest path in unweighted graphs. Foundation for many other algorithms.

TimeO(V + E)
SpaceO(V)
static int[][] bfs(List<List<Integer>> adj, int start) {
    int n = adj.size();
    int[] dist = new int[n];
    int[] parent = new int[n];
    Arrays.fill(dist, -1);
    Arrays.fill(parent, -1);
    dist[start] = 0;
    Deque<Integer> q = new ArrayDeque<>();
    q.offer(start);
    while (!q.isEmpty()) {
        int u = q.poll();
        for (int v : adj.get(u)) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                parent[v] = u;
                q.offer(v);
            }
        }
    }
    return new int[][]{dist, parent};
}

Use cases: shortest path (unweighted), level-order traversal, finding connected components, web crawling, social network "degrees of separation", garbage collection (mark phase), finding cycles in undirected graphs.

6. Depth-First Search (DFS)

Explores as far as possible before backtracking. Uses a stack (or recursion). Underpins topological sort, SCCs, cycle detection, and bridges.

TimeO(V + E)
SpaceO(V)
static List<Integer> dfs(List<List<Integer>> adj, int start) {
    boolean[] visited = new boolean[adj.size()];
    List<Integer> order = new ArrayList<>();
    dfsVisit(adj, start, visited, order);
    return order;
}

static void dfsVisit(List<List<Integer>> adj, int u,
                     boolean[] visited, List<Integer> order) {
    visited[u] = true;
    order.add(u);
    for (int v : adj.get(u))
        if (!visited[v]) dfsVisit(adj, v, visited, order);
}

// Iterative version (avoid stack overflow)
static void dfsIter(List<List<Integer>> adj, int start) {
    boolean[] visited = new boolean[adj.size()];
    Deque<Integer> stack = new ArrayDeque<>();
    stack.push(start);
    visited[start] = true;
    while (!stack.isEmpty()) {
        int u = stack.pop();
        for (int v : adj.get(u)) {
            if (!visited[v]) {
                visited[v] = true;
                stack.push(v);
            }
        }
    }
}

DFS Edge Classification (in DAG/digraph)

  • Tree edge: discovered new vertex.
  • Back edge: to ancestor (indicates cycle).
  • Forward edge: to descendant.
  • Cross edge: between different subtrees.

7. Topological Sort

Linear ordering of a DAG's vertices such that for every edge u → v, u appears before v. Used to schedule tasks with dependencies.

Kahn's Algorithm (BFS-based)

static List<Integer> topoKahn(int n, int[][] edges) {
    List<List<Integer>> adj = new ArrayList<>();
    for (int i = 0; i < n; i++) adj.add(new ArrayList<>());
    int[] indeg = new int[n];
    for (int[] e : edges) {
        adj.get(e[0]).add(e[1]);
        indeg[e[1]]++;
    }
    Deque<Integer> q = new ArrayDeque<>();
    for (int i = 0; i < n; i++) if (indeg[i] == 0) q.offer(i);
    List<Integer> order = new ArrayList<>();
    while (!q.isEmpty()) {
        int u = q.poll();
        order.add(u);
        for (int v : adj.get(u))
            if (--indeg[v] == 0) q.offer(v);
    }
    return order.size() == n ? order : null;   // null means cycle
}

DFS-based Topological Sort

static List<Integer> topoDfs(int n, List<List<Integer>> adj) {
    boolean[] visited = new boolean[n];
    Deque<Integer> order = new ArrayDeque<>();
    for (int i = 0; i < n; i++)
        if (!visited[i]) topoVisit(i, adj, visited, order);
    return new ArrayList<>(order);   // pushed in reverse-finish order
}

static void topoVisit(int u, List<List<Integer>> adj,
                      boolean[] visited, Deque<Integer> order) {
    visited[u] = true;
    for (int v : adj.get(u))
        if (!visited[v]) topoVisit(v, adj, visited, order);
    order.push(u);
}

Real-world: course prerequisites, build systems (Make, Bazel), Excel formula recalc, package installation, instruction scheduling in compilers.

8. Disjoint Set Union (Union-Find)

A data structure for tracking elements partitioned into disjoint sets. Two operations: find(x) returns set representative; union(x, y) merges sets. Lightning fast with two optimizations: path compression and union by rank/size.

Amortized opO(α(n))
α(n)≤ 4 for any practical n
class DSU {
    int[] parent, rank, size;
    int components;

    DSU(int n) {
        parent = new int[n];
        rank = new int[n];
        size = new int[n];
        components = n;
        for (int i = 0; i < n; i++) { parent[i] = i; size[i] = 1; }
    }

    int find(int x) {
        while (parent[x] != x) {
            parent[x] = parent[parent[x]];   // path compression
            x = parent[x];
        }
        return x;
    }

    boolean union(int x, int y) {
        int rx = find(x), ry = find(y);
        if (rx == ry) return false;
        if (rank[rx] < rank[ry]) { int t = rx; rx = ry; ry = t; }
        parent[ry] = rx;
        size[rx] += size[ry];
        if (rank[rx] == rank[ry]) rank[rx]++;
        components--;
        return true;
    }

    boolean connected(int x, int y) { return find(x) == find(y); }
}
Where DSU shines: Kruskal's MST, detecting cycles in undirected graphs, dynamic connectivity, image processing (connected pixel regions), Kruskal-style clustering, percolation simulations, and offline LCA queries.

Variants

  • Weighted DSU: tracks relative weights between elements (used in equation systems).
  • Persistent DSU: supports rollback for offline algorithms.
  • DSU on tree (small-to-large): classic competitive-programming technique.

9. Strongly Connected Components (SCC)

In a digraph, an SCC is a maximal subgraph where every vertex is reachable from every other. Two famous algorithms:

Kosaraju's Algorithm

  1. DFS on original graph, push vertices onto stack in finish order.
  2. Reverse the graph.
  3. Pop from stack, run DFS on reversed graph — each tree is an SCC.
static int[] kosaraju(int n, List<List<Integer>> adj) {
    List<List<Integer>> rev = new ArrayList<>();
    for (int i = 0; i < n; i++) rev.add(new ArrayList<>());
    for (int u = 0; u < n; u++)
        for (int v : adj.get(u)) rev.get(v).add(u);

    boolean[] visited = new boolean[n];
    Deque<Integer> order = new ArrayDeque<>();
    for (int i = 0; i < n; i++)
        if (!visited[i]) dfs1(i, adj, visited, order);

    int[] comp = new int[n];
    Arrays.fill(comp, -1);
    int c = 0;
    while (!order.isEmpty()) {
        int u = order.pop();
        if (comp[u] == -1) { dfs2(u, c, rev, comp); c++; }
    }
    return comp;
}

static void dfs1(int u, List<List<Integer>> adj,
                 boolean[] visited, Deque<Integer> order) {
    visited[u] = true;
    for (int v : adj.get(u))
        if (!visited[v]) dfs1(v, adj, visited, order);
    order.push(u);
}

static void dfs2(int u, int c, List<List<Integer>> rev, int[] comp) {
    comp[u] = c;
    for (int v : rev.get(u))
        if (comp[v] == -1) dfs2(v, c, rev, comp);
}

Tarjan's Algorithm

Single-pass DFS using discovery times and a "low-link" value; identifies SCCs with one stack. O(V + E).

Applications: 2-SAT solver, condensation graphs (DAG of SCCs), deadlock detection, recommendation systems, finding cycles in dependency graphs.

10. Bridges & Articulation Points

Bridge: an edge whose removal disconnects the graph. Articulation point: a vertex whose removal disconnects the graph. Found via DFS in O(V + E) using discovery times disc[u] and lowest reachable ancestor low[u].

class BridgeFinder {
    int[] disc, low, parent;
    int timer = 0;
    List<int[]> result = new ArrayList<>();
    List<List<Integer>> adj;

    List<int[]> bridges(int n, List<List<Integer>> adj) {
        this.adj = adj;
        disc = new int[n]; low = new int[n]; parent = new int[n];
        Arrays.fill(disc, -1); Arrays.fill(parent, -1);
        for (int i = 0; i < n; i++) if (disc[i] == -1) dfs(i);
        return result;
    }

    void dfs(int u) {
        disc[u] = low[u] = timer++;
        for (int v : adj.get(u)) {
            if (disc[v] == -1) {
                parent[v] = u;
                dfs(v);
                low[u] = Math.min(low[u], low[v]);
                if (low[v] > disc[u]) result.add(new int[]{u, v});   // bridge
            } else if (v != parent[u]) {
                low[u] = Math.min(low[u], disc[v]);
            }
        }
    }
}

Real-world: identifying single points of failure in networks, vulnerability analysis in power grids, critical roads in transportation.

11. Dijkstra's Algorithm

Finds shortest paths from a source to all vertices in a graph with non-negative edge weights. Greedy: repeatedly extract the closest unsettled vertex and relax its edges.

Binary heapO((V + E) log V)
Fibonacci heapO(E + V log V)
ArrayO(V²)
static long[] dijkstra(int n, List<List<int[]>> adj, int src) {
    long[] dist = new long[n];
    int[] parent = new int[n];
    Arrays.fill(dist, Long.MAX_VALUE);
    Arrays.fill(parent, -1);
    dist[src] = 0;
    // PQ entries: long[]{distance, node}
    PriorityQueue<long[]> pq = new PriorityQueue<>(
        (a, b) -> Long.compare(a[0], b[0]));
    pq.offer(new long[]{0, src});
    while (!pq.isEmpty()) {
        long[] cur = pq.poll();
        long d = cur[0];
        int u = (int) cur[1];
        if (d > dist[u]) continue;          // stale entry
        for (int[] e : adj.get(u)) {
            int v = e[0], w = e[1];
            long nd = d + w;
            if (nd < dist[v]) {
                dist[v] = nd;
                parent[v] = u;
                pq.offer(new long[]{nd, v});
            }
        }
    }
    return dist;
}
Caveat: Dijkstra fails with negative edges because once we settle a node we assume no shorter path exists — a negative edge could violate that. Use Bellman-Ford for negative weights.

Real-world: Google Maps / Waze routing, IP routing (OSPF, IS-IS), robotics path planning, network latency optimization, currency arbitrage (with negation).

12. Bellman-Ford Algorithm

Computes shortest paths from a source even with negative edge weights. Detects negative cycles reachable from the source. Slower than Dijkstra but more general.

TimeO(V · E)
SpaceO(V)
static long[] bellmanFord(int n, int[][] edges, int src) {
    long[] dist = new long[n];
    Arrays.fill(dist, Long.MAX_VALUE);
    dist[src] = 0;
    for (int i = 0; i < n - 1; i++) {
        boolean updated = false;
        for (int[] e : edges) {
            int u = e[0], v = e[1], w = e[2];
            if (dist[u] != Long.MAX_VALUE && dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                updated = true;
            }
        }
        if (!updated) break;           // early termination
    }
    // Negative cycle detection
    for (int[] e : edges) {
        int u = e[0], v = e[1], w = e[2];
        if (dist[u] != Long.MAX_VALUE && dist[u] + w < dist[v])
            return null;               // negative cycle reachable from src
    }
    return dist;
}

Why V−1 iterations? The shortest path uses at most V−1 edges (no cycles needed). After V−1 relaxations, if anything still improves, a negative cycle exists.

SPFA (Shortest Path Faster Algorithm)

A queue-based optimization of Bellman-Ford. Average O(kE) where k is small in practice, but worst case still O(VE).

Real-world: RIP routing protocol, currency arbitrage detection, financial modeling with rebates, parsing expression grammars with negative weights.

13. Floyd-Warshall (All-Pairs Shortest Paths)

Computes shortest paths between every pair of vertices. Dynamic programming over intermediate vertices. Handles negative weights (no negative cycles).

TimeO(V³)
SpaceO(V²)
static long[][] floydWarshall(int n, long[][] weight) {
    long[][] dist = new long[n][n];
    for (int i = 0; i < n; i++)
        dist[i] = weight[i].clone();
    for (int k = 0; k < n; k++)
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++)
                if (dist[i][k] != Long.MAX_VALUE
                    && dist[k][j] != Long.MAX_VALUE
                    && dist[i][k] + dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k] + dist[k][j];
    // Negative cycle if any dist[i][i] < 0
    return dist;
}

The DP insight: dist[i][j] after iteration k is the shortest path using only vertices {0..k} as intermediates.

Use cases: transitive closure, finding diameter of a small graph, Warshall's algorithm for reachability, dense graphs where V is small (≤ 500).

14. Johnson's Algorithm

All-pairs shortest paths for sparse graphs with possibly negative weights. Cleverly combines Bellman-Ford and Dijkstra.

  1. Add a dummy vertex q with 0-weight edges to every vertex.
  2. Run Bellman-Ford from q to get h(v); detect negative cycles.
  3. Reweight every edge (u, v, w) to w + h(u) − h(v) (now non-negative).
  4. Run Dijkstra from each vertex on the reweighted graph.
  5. Restore original distances: d(u, v) = d'(u, v) − h(u) + h(v).
TimeO(V² log V + VE)

Beats Floyd-Warshall when E ≪ V².

15. A* Search

Best-first search using f(n) = g(n) + h(n) where g is cost so far and h is an admissible heuristic (never overestimates). Optimal and complete given an admissible heuristic.

interface Neighbors<T> { List<Edge<T>> of(T node); }
interface Heuristic<T> { double estimate(T node); }
record Edge<T>(T to, double weight) {}

static <T> List<T> aStar(T start, T goal,
                          Neighbors<T> neighbors, Heuristic<T> h) {
    Map<T, Double> g = new HashMap<>();
    Map<T, T> parent = new HashMap<>();
    g.put(start, 0.0);
    parent.put(start, null);
    PriorityQueue<Object[]> pq = new PriorityQueue<>(
        (a, b) -> Double.compare((double) a[0], (double) b[0]));
    pq.offer(new Object[]{h.estimate(start), 0.0, start});
    while (!pq.isEmpty()) {
        Object[] cur = pq.poll();
        double gu = (double) cur[1];
        @SuppressWarnings("unchecked") T u = (T) cur[2];
        if (u.equals(goal)) return reconstruct(parent, u);
        if (gu > g.get(u)) continue;
        for (Edge<T> e : neighbors.of(u)) {
            double ng = gu + e.weight();
            if (ng < g.getOrDefault(e.to(), Double.POSITIVE_INFINITY)) {
                g.put(e.to(), ng);
                parent.put(e.to(), u);
                pq.offer(new Object[]{ng + h.estimate(e.to()), ng, e.to()});
            }
        }
    }
    return null;
}

Heuristics: Euclidean distance, Manhattan distance (grids), straight-line distance, learned heuristics (ML).

Real-world: game pathfinding (NPCs in video games), GPS navigation with ETA, robot motion planning, puzzle solvers (8-puzzle, Rubik's cube).

Variants

  • IDA*: iterative deepening A* — memory efficient.
  • D* / D* Lite: dynamic replanning when graph changes (Mars rovers).
  • Bidirectional Dijkstra / A*: search from both ends; meet in middle.

16. Kruskal's Minimum Spanning Tree

Greedy: sort edges, add the cheapest edge that doesn't form a cycle (checked via DSU). Produces an MST: connects all vertices with minimum total weight.

TimeO(E log E)
Bottleneckedge sort
// edges: int[] {u, v, weight}
static int kruskal(int n, int[][] edges, List<int[]> mstOut) {
    Arrays.sort(edges, (a, b) -> Integer.compare(a[2], b[2]));
    DSU dsu = new DSU(n);
    int total = 0;
    for (int[] e : edges) {
        if (dsu.union(e[0], e[1])) {
            mstOut.add(e);
            total += e[2];
            if (mstOut.size() == n - 1) break;
        }
    }
    return total;
}

Real-world: network design (laying cables, pipelines), clustering (single-link), approximate TSP, image segmentation.

17. Prim's Minimum Spanning Tree

Grow the MST one vertex at a time from a starting vertex, always adding the cheapest edge crossing the boundary. Better for dense graphs.

Binary heapO((V + E) log V)
Fibonacci heapO(E + V log V)
static int prim(int n, List<List<int[]>> adj, List<int[]> mstOut) {
    boolean[] visited = new boolean[n];
    // PQ entries: int[]{weight, node, parent}
    PriorityQueue<int[]> pq = new PriorityQueue<>(
        (a, b) -> Integer.compare(a[0], b[0]));
    pq.offer(new int[]{0, 0, -1});
    int total = 0, taken = 0;
    while (!pq.isEmpty() && taken < n) {
        int[] cur = pq.poll();
        int w = cur[0], u = cur[1], p = cur[2];
        if (visited[u]) continue;
        visited[u] = true;
        total += w;
        taken++;
        if (p != -1) mstOut.add(new int[]{p, u, w});
        for (int[] e : adj.get(u)) {
            int v = e[0], ew = e[1];
            if (!visited[v]) pq.offer(new int[]{ew, v, u});
        }
    }
    return total;
}
Kruskal vs Prim: Both give an MST. Use Kruskal when edges are pre-sorted or graph is sparse and given as edge list. Use Prim with a heap on dense adjacency-list graphs. Borůvka's is a third option, parallelizable in O(E log V).

18. Maximum Flow (Ford-Fulkerson, Edmonds-Karp, Dinic)

Given a directed graph with edge capacities, find the maximum flow from source s to sink t. Repeatedly find augmenting paths and push flow until none remain.

Ford-Fulkerson with BFS (Edmonds-Karp)

TimeO(V · E²)
static int edmondsKarp(int n, int[][] capacity, int s, int t) {
    int[][] flow = new int[n][n];
    int maxFlow = 0;
    while (true) {
        int[] parent = new int[n];
        Arrays.fill(parent, -1);
        parent[s] = s;
        Deque<Integer> q = new ArrayDeque<>();
        q.offer(s);
        while (!q.isEmpty() && parent[t] == -1) {
            int u = q.poll();
            for (int v = 0; v < n; v++) {
                if (parent[v] == -1 && capacity[u][v] - flow[u][v] > 0) {
                    parent[v] = u;
                    q.offer(v);
                }
            }
        }
        if (parent[t] == -1) break;
        // bottleneck
        int pathFlow = Integer.MAX_VALUE;
        for (int v = t; v != s; v = parent[v]) {
            int u = parent[v];
            pathFlow = Math.min(pathFlow, capacity[u][v] - flow[u][v]);
        }
        for (int v = t; v != s; v = parent[v]) {
            int u = parent[v];
            flow[u][v] += pathFlow;
            flow[v][u] -= pathFlow;
        }
        maxFlow += pathFlow;
    }
    return maxFlow;
}

Dinic's Algorithm

Builds a layered graph via BFS, then finds blocking flows via DFS. O(V² · E) general, O(E · √V) on unit-capacity / bipartite graphs.

Max-Flow Min-Cut Theorem

The maximum flow equals the minimum capacity of any s-t cut. This duality unlocks countless problem reductions.

Real-world: bipartite matching, image segmentation, network reliability, baseball elimination, project selection, airline scheduling, traffic optimization.

19. Bipartite Matching

In a bipartite graph (two sets U and V), find the largest set of edges with no shared endpoints. Reducible to max flow with a super-source and super-sink.

Hopcroft-Karp Algorithm

Augments along multiple shortest paths simultaneously. O(E · √V) — best general bipartite matching algorithm.

Hungarian Algorithm

For weighted bipartite matching (assignment problem): find perfect matching of minimum (or maximum) total weight. O(V³).

Real-world: assigning workers to jobs, ride-share dispatch (Uber/Lyft), kidney exchange programs, ad auctions, scheduling exams to rooms.

20. Traveling Salesman Problem (TSP)

Find the shortest tour visiting every vertex exactly once and returning to start. NP-hard.

Held-Karp DP

Bitmask DP over subsets: dp[mask][i] = shortest path visiting set mask ending at i. O(2ⁿ · n²) time, O(2ⁿ · n) space — feasible up to ~20 cities.

static int tsp(int[][] dist) {
    int n = dist.length;
    int INF = Integer.MAX_VALUE / 2;
    int[][] dp = new int[1 << n][n];
    for (int[] row : dp) Arrays.fill(row, INF);
    dp[1][0] = 0;
    for (int mask = 0; mask < (1 << n); mask++) {
        for (int u = 0; u < n; u++) {
            if (dp[mask][u] == INF || ((mask >> u) & 1) == 0) continue;
            for (int v = 0; v < n; v++) {
                if (((mask >> v) & 1) == 1) continue;
                int nm = mask | (1 << v);
                int cost = dp[mask][u] + dist[u][v];
                if (cost < dp[nm][v]) dp[nm][v] = cost;
            }
        }
    }
    int best = INF, full = (1 << n) - 1;
    for (int u = 1; u < n; u++)
        best = Math.min(best, dp[full][u] + dist[u][0]);
    return best;
}

Approximations

  • Christofides: 1.5-approximation for metric TSP using MST + minimum-weight matching.
  • 2-opt / 3-opt / Lin-Kernighan: local search heuristics, near-optimal in practice.
  • Genetic algorithms, simulated annealing, ACO: metaheuristics.

Real-world: delivery routing (UPS ORION saves ~$300M/yr), PCB drilling order, DNA sequencing, telescope scheduling.

Hierholzer's Algorithm (Eulerian circuit)

static List<Integer> hierholzer(List<Deque<Integer>> adj, int start) {
    Deque<Integer> stack = new ArrayDeque<>();
    List<Integer> circuit = new ArrayList<>();
    stack.push(start);
    while (!stack.isEmpty()) {
        int u = stack.peek();
        if (!adj.get(u).isEmpty()) {
            stack.push(adj.get(u).pop());
        } else {
            circuit.add(stack.pop());
        }
    }
    Collections.reverse(circuit);
    return circuit;
}

Famous example: the Königsberg bridges problem — Euler founded graph theory by proving no such walk exists.

Real-world: DNA fragment assembly (de Bruijn graphs), Chinese Postman problem (street sweeping, snowplow routing), CNC machine tool paths.

22. PageRank

The algorithm that powered Google. Computes a stationary distribution of a random walk on the web graph with teleportation.

PR(u) = (1−d)/N + d · Σ PR(v)/L(v) over inbound links v → u, where d ≈ 0.85 is the damping factor.

static double[] pagerank(List<List<Integer>> adj, int n,
                         double d, int iters, double tol) {
    double[] pr = new double[n];
    Arrays.fill(pr, 1.0 / n);
    int[] out = new int[n];
    for (int i = 0; i < n; i++) out[i] = adj.get(i).size();

    for (int it = 0; it < iters; it++) {
        double[] next = new double[n];
        Arrays.fill(next, (1 - d) / n);
        for (int u = 0; u < n; u++) {
            if (out[u] == 0) {
                double share = pr[u] / n;            // dangling redistribution
                for (int v = 0; v < n; v++) next[v] += d * share;
            } else {
                double share = d * pr[u] / out[u];
                for (int v : adj.get(u)) next[v] += share;
            }
        }
        double delta = 0;
        for (int i = 0; i < n; i++) delta += Math.abs(pr[i] - next[i]);
        pr = next;
        if (delta < tol) break;
    }
    return pr;
}

Variants: Personalized PageRank (recommender systems), HITS (Hubs and Authorities), TrustRank (spam detection), TextRank (keyword extraction).

Decision Tree: Which Algorithm?

  • Need all reachable nodes? → BFS or DFS.
  • Need shortest path?
    • Unweighted → BFS.
    • Non-negative weights, single source → Dijkstra.
    • Negative weights, single source → Bellman-Ford.
    • All pairs, dense → Floyd-Warshall.
    • All pairs, sparse → Johnson.
    • With heuristic → A*.
  • Need cheapest connection of all nodes? → MST (Kruskal/Prim).
  • Need ordering with dependencies? → Topological sort (DAG).
  • Need to track groups/components? → DSU.
  • Need maximum throughput? → Max flow (Dinic).
  • Need to pair items optimally? → Bipartite matching / Hungarian.
  • Need to find cycles or critical edges? → DFS-based (Tarjan, low-link).
Final advice: 90% of "graph problems" you'll see in interviews and real systems are solved with BFS, DFS, Dijkstra, topological sort, DSU, or MST. Master those six and you'll handle the vast majority. Save the heavyweights (max flow, Hungarian, NP-hard) for when you genuinely need them.
